## Highlighted Projects

### Fortnite Graph Loot Engine
The <a href="https://github.com/garrettbovo/GraphLoot-Engine">Fortnite Graph Loot Engine</a> is a C++17 simulation engine that models traversal across a weighted graph of Points of Interest (POIs). It integrates Dijkstra’s shortest-path algorithm with a probability-driven loot system to support both real-time gameplay and large-scale statistical analysis (1,000,000+ runs). The system is fully data-driven, loading map topology and loot configurations from CSV files, and is built with a CMake-based build system, automated testing, and CI validation.

### Pokémon Go Game Simulation
A <a href="https://github.com/garrettbovo/Pokemon-Go-Game-Simulation">C-based event-driven system</a> that models player interactions, encounter generation, and inventory state transitions within a dynamic environment. The system integrates multiple core data structures, including a trie for efficient name-based lookup and a doubly linked list for managing captured entities. Implements modular manager structures using function pointers to encapsulate menu handling, list operations, and trie functionality, enabling flexible and extensible system behavior. Features manual memory management, pointer-based data traversal, and file I/O pipelines for persistent output generation and state reporting.

### Student Registration System Simulation
A <a href="https://github.com/garrettbovo/Student-Registration-System-Simulation">C-based data management system</a> that models student enrollment workflows, supporting dynamic insertion, deletion, modification, and querying of structured records. The system enforces data integrity through validated input pipelines and uses a trie-based index to ensure uniqueness of student identifiers (NetIDs). Implements a doubly linked list for efficient record management, including sorting, filtering, and in-place transformations. Utilizes modular manager structures with function pointers to encapsulate menu handling, list operations, and search functionality, enabling extensible system behavior. Emphasizes manual memory management, pointer-based traversal, and file output for persistent state reporting.
